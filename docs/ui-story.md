# 佛经 App UI Story

## 1. Story 目标

这份 UI Story 用一个完整日常场景，把首版 App 的关键页面串起来。

设计主线：

用户不是来探索复杂功能，而是在一天中安静地完成读经、诵经和回向。

体验关键词：

- 一眼知道今天要做什么。
- 两步以内开始功课。
- 阅读过程中不被打扰。
- 完成后自然回向。
- 所有记录安静地保存，不制造压力。

## 2. 角色设定

用户：日常修行者

背景：

- 每天早上或晚上固定诵心经、大悲咒、佛号。
- 希望手机 App 像一本干净经书。
- 不喜欢广告、排行榜、弹窗、复杂社交。
- 偶尔会查询经文来源和注释。

今日功课：

- 心经 1 遍。
- 大悲咒 3 遍。
- 阿弥陀佛 108 声。

## 3. 体验节奏

```text
打开 App
  ↓
今日页看到功课
  ↓
点击开始今日功课
  ↓
进入心经阅读/诵读
  ↓
完成后进入下一项
  ↓
大悲咒诵读
  ↓
佛号计数
  ↓
回向
  ↓
今日页显示完成
```

## 4. Scene 1：打开 App

### 用户意图

用户想快速开始今天的功课，不想先浏览大量内容。

### 界面状态

```text
┌──────────────────────────┐
│ 今日                     │
│ 乙巳年 五月十八 · 晚课    │
│                          │
│ 今日功课                 │
│ ┌──────────────────────┐ │
│ │ 心经          未开始  │ │
│ │ 大悲咒        未开始  │ │
│ │ 阿弥陀佛      0/108   │ │
│ └──────────────────────┘ │
│                          │
│ [开始今日功课]            │
│                          │
│ 最近阅读                 │
│ 金刚经 · 第十三品         │
│                          │
│ 今日一句                 │
│ 照见五蕴皆空             │
└──────────────────────────┘
```

### UX 要点

- “开始今日功课”是唯一强按钮。
- 今日功课不是卡片堆叠，而是一块安静的进度区域。
- 最近阅读和今日一句是辅助入口，视觉权重低于功课。

### 微交互

- 如果今天有佛教节日，在日期下显示一行提示。
- 如果功课全部完成，主按钮变为“查看今日记录”。
- 如果用户没有设置功课，主按钮变为“设置日课”。

## 5. Scene 2：开始心经

### 用户意图

用户点击开始后，希望直接进入第一项，不再选择。

### 界面状态

```text
┌──────────────────────────┐
│ < 心经             设置   │
│                          │
│ 观自在菩萨，行深般若波罗蜜│
│ 多时，照见五蕴皆空，度一切│
│ 苦厄。                   │
│                          │
│ 舍利子，色不异空，空不异色│
│ 色即是空，空即是色。      │
│                          │
│                          │
│ ┌──────────────────────┐ │
│ │ 00:00    ▶      1/14 │ │
│ └──────────────────────┘ │
└──────────────────────────┘
```

### UX 要点

- 阅读页不显示底部 Tab，避免打扰。
- 正文留白充足。
- 底部播放器轻量悬浮，默认不遮挡正文。
- 播放按钮居中，目录和书签等放在次级位置。

### 微交互

- 点击播放后，当前句柔和高亮。
- 自动滚动保持当前句位于屏幕中部偏上。
- 播放完成后出现轻提示：“心经已完成 1 遍”。

## 6. Scene 3：阅读设置

### 用户意图

用户觉得字号略小，想调大并打开繁体。

### 界面状态

```text
┌──────────────────────────┐
│ 阅读设置                 │
│                          │
│ 字号                     │
│ 小 ───────●──── 大        │
│                          │
│ 字体                     │
│ [简体] [繁体]             │
│                          │
│ 显示                     │
│ 拼音        ○             │
│ 注释        ●             │
│ 夜间模式    ○             │
│ 自动滚动    ●             │
└──────────────────────────┘
```

### UX 要点

- 设置作为 Sheet 出现，不离开阅读上下文。
- 设置变更即时作用于正文。
- 只放阅读相关设置，不混入账号、会员、隐私等内容。

## 7. Scene 4：大悲咒诵读

### 用户意图

心经完成后，系统自动进入下一项大悲咒。

### 界面状态

```text
┌──────────────────────────┐
│ < 大悲咒           2/3 遍 │
│                          │
│ 南无喝啰怛那哆啰夜耶。    │
│ 南无阿唎耶。              │
│ 婆卢羯帝烁钵啰耶。        │
│ 菩提萨埵婆耶。            │
│                          │
│ 当前第 2 遍               │
│                          │
│ ┌──────────────────────┐ │
│ │ 01:18   暂停    循环  │ │
│ └──────────────────────┘ │
└──────────────────────────┘
```

### UX 要点

- 多遍诵读时，遍数显示在标题区域。
- 不用夸张完成动画。
- 完成一遍后只做轻微状态更新。

### 微交互

- 单遍完成后，遍数从 1/3 更新为 2/3。
- 如果用户暂停，下方显示“继续诵读”。
- 如果用户退出，今日页保留当前进度。

## 8. Scene 5：佛号计数

### 用户意图

用户想专注念佛，每念一声轻点一次。

### 界面状态

```text
┌──────────────────────────┐
│ < 阿弥陀佛               │
│                          │
│ 今日目标                 │
│ 108 声                   │
│                          │
│            36            │
│                          │
│        [  记一声  ]       │
│                          │
│ 木鱼  开                 │
│ 引磬  关                 │
│                          │
│ [完成本项]                │
└──────────────────────────┘
```

### UX 要点

- 计数数字大，但不做炫耀式视觉。
- “记一声”按钮足够大，适合闭眼或半专注点击。
- 木鱼和引磬是辅助开关，不能抢占视觉。

### 微交互

- 点击时有轻微触觉反馈。
- 开启木鱼后，每次点击播放短音。
- 长按数字可撤销上一次计数。
- 达到 108 后按钮变为“完成本项”。

## 9. Scene 6：回向

### 用户意图

功课完成后，用户想用常用回向文结束今日功课。

### 界面状态

```text
┌──────────────────────────┐
│ 回向                     │
│                          │
│ 今日已完成               │
│ 心经 1 遍                │
│ 大悲咒 3 遍              │
│ 阿弥陀佛 108 声           │
│                          │
│ 回向文                   │
│ ┌──────────────────────┐ │
│ │ 愿以此功德，庄严佛净土 │ │
│ │ 上报四重恩，下济三途苦 │ │
│ │ 若有见闻者，悉发菩提心 │ │
│ │ 尽此一报身，同生极乐国 │ │
│ └──────────────────────┘ │
│                          │
│ 回向对象                 │
│ [法界众生] [家人] [自定义]│
│                          │
│ [完成并保存]              │
└──────────────────────────┘
```

### UX 要点

- 回向页不应像复杂表单。
- 默认选择通用回向文和法界众生。
- 自定义对象是可选项，不打断流程。

### 微交互

- 点击模板切换时保留用户自定义内容。
- 保存后显示短提示：“今日功课已保存”。
- 返回今日页。

## 10. Scene 7：完成后的今日页

### 用户意图

用户想确认今日功课完成。

### 界面状态

```text
┌──────────────────────────┐
│ 今日                     │
│ 乙巳年 五月十八 · 已完成  │
│                          │
│ 今日功课                 │
│ ┌──────────────────────┐ │
│ │ ✓ 心经        1/1 遍 │ │
│ │ ✓ 大悲咒      3/3 遍 │ │
│ │ ✓ 阿弥陀佛  108/108 │ │
│ └──────────────────────┘ │
│                          │
│ [查看今日记录]            │
│                          │
│ 回向                     │
│ 愿以此功德，庄严佛净土... │
└──────────────────────────┘
```

### UX 要点

- 完成状态明确，但不做过度庆祝。
- 保留回向摘要，让一天的修行有闭环。
- 用户可以查看记录，但不会被推动去分享。

## 11. 视觉演进建议

### 第一版

低保真实现即可：

- SwiftUI 原生列表和导航。
- 自定义阅读正文排版。
- 简洁底部播放器。
- 基础设置 Sheet。

### 第二版

打磨质感：

- 更好的纸张背景。
- 更细致的段落样式。
- 阅读页底部播放器加入轻材质效果。
- 音频逐句高亮更自然。

### 第三版

加入学习辅助：

- 术语解释。
- 经文导读。
- 阅读路径。
- 基于出处的 AI 问答。

## 12. 设计风险

### 风险 1：功能太多，首页变乱

解决：

- 首页只服务今日功课。
- 经文探索放到经藏页。
- 设置和历史放到我的页。

### 风险 2：阅读器被工具栏干扰

解决：

- 工具栏默认轻量。
- 滚动时可自动弱化。
- 播放器只占底部一小块。

### 风险 3：日课像打卡软件

解决：

- 不显示连续天数作为主指标。
- 不做排行榜。
- 不用强烈庆祝动画。
- 用“今日记录”而不是“成就”。

### 风险 4：佛教元素装饰过重

解决：

- 用文字、留白、纸张感表达气质。
- 少用佛像、莲花、金光背景。
- 把庄重感放在排版和节奏里。
