# 净诵发版检查清单

## 发布原则

每次发布都优先保证日课进度、阅读进度、回向记录和本地隐私承诺可靠。小版本可以只修一个问题，但必须跑完固定验证。

## 必须保留

- 今日功课：心经、大悲咒、阿弥陀佛计数。
- 经藏：可浏览正式经文，未接入全文的经文只能查看详情，不能进入占位阅读。
- 阅读器：字号、简繁、拼音、注释、夜间模式、书签、回到开头、系统朗读、单句循环、完成。
- 回向：完成后保存当日回向，避免当天重复保存。
- 我的：统计、最近回向记录、最近日课、反馈入口、重置今日功课。

## 每次发版前

1. 更新版本号和构建号：
   - `MARKETING_VERSION`：用户可见版本，例如 `1.0.2`
   - `CURRENT_PROJECT_VERSION`：每次上传递增，例如 `4`
2. 运行 XcodeGen：

```bash
xcodegen generate
```

3. 运行自动化测试：

```bash
xcodebuild test -project FoJing.xcodeproj -scheme FoJingApp -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
```

4. 归档上传 TestFlight，等待 build 处理完成。
5. 用 TestFlight 真机安装当前 build 后完整回归。
6. App Store Connect 绑定正确 build，填写 What's New，提交审核。

## 真机手测路径

1. 今日页进入心经，完成。
2. 彻底退出 App，重新打开，确认心经仍完成。
3. 再次彻底退出 App，重新打开，确认心经仍完成。
4. 继续大悲咒，完成。
5. 阿弥陀佛计数到目标，确认进入回向按钮可用。
6. 进入回向并保存。
7. 我的页确认最近日课和最近回向记录存在。
8. 点反馈与支持，确认能打开邮件并预填版本号。
9. 重置今日功课，确认今日页恢复。
10. 大字号模式检查今日页、阅读页播放栏、回向页、我的页。

## App Store 检查

- What's New 只写当前版本真实变化。
- Promotional Text 小修复版通常不改，除非要更新长期卖点。
- 支持 URL 必须是可公开访问的支持页面，页面内包含 `foxnet2000@gmail.com`。
- 隐私政策 URL 必须可公开访问，并与 App 内“不联网、不收集数据”的实际行为一致。
- 截图来自当前发布版本，不展示未发布功能。
- 审核备注说明：App 不需要登录，不联网；数据保存在本机；朗读使用 iOS 系统 TTS；木鱼和引磬为本地合成音效。

## 后续版本候选

- 更多经文全文接入。
- 书签列表和书签详情。
- 回向记录详情页。
- 日课自定义。
- 音频真实文件和精确时间轴；1.0 先使用系统中文 TTS 朗读正式经文。
- 多版本对照、抄经、笔记、高亮。
