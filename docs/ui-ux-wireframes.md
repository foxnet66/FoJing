# 佛经 App UI/UX 线框文档

## 1. 设计方向

采用“清净阅读型”为主，“日课流程”为辅。

视觉目标：

- 安静。
- 可信。
- 克制。
- 易读。
- 长时间使用不疲劳。

不要做成：

- 功能入口堆满的工具箱。
- 强打卡 App。
- 社交社区。
- 宗教装饰过重的页面。
- 广告和会员入口明显的商业 App。

## 2. 视觉基调

### 2.1 色彩

建议使用低饱和、接近纸张和墨色的配色：

- 背景：纸白、暖白、浅米色。
- 正文：墨黑、深灰。
- 辅助：竹青、沉香灰。
- 强调：淡金、朱砂红少量使用。

使用原则：

- 正文页以阅读舒适为第一优先。
- 金色和红色只用于状态、节日、重点，不做大面积背景。
- 夜间模式使用暖黑，不使用纯黑。

### 2.2 字体

中文正文：

- 优先使用系统中文字体。
- 正文字号默认 20-22pt。
- 行距明显大于普通新闻阅读。
- 偈颂和咒语可使用更稳定的分行。

界面文字：

- 使用系统 San Francisco / PingFang。
- 标题不要过大。
- 按钮文案短而明确。

### 2.3 图标

使用 iOS 原生 SF Symbols：

- book.closed
- text.book.closed
- play.circle
- pause.circle
- bookmark
- magnifyingglass
- list.bullet
- moon
- speaker.wave.2
- bell
- checkmark.circle

图标作为辅助，不喧宾夺主。

## 3. App 导航结构

底部 Tab：

1. 今日
2. 经藏
3. 诵持
4. 我的

iOS 结构建议：

- AppShell: TabView。
- 每个 Tab 内部使用 NavigationStack。
- Sheet 用于设置面板、回向编辑、功课编辑。
- 阅读器内的播放条固定在底部。

## 4. 今日页

### 4.1 页面目标

用户打开 App 后，马上知道：

- 今天有没有特别日子。
- 今天要完成什么功课。
- 上次读到哪里。
- 如何开始。

### 4.2 线框

```text
┌──────────────────────────┐
│ 今日                     │
│ 乙巳年 五月十八          │
│                          │
│ 若有佛教节日，显示一行    │
│                          │
│ 今日功课                 │
│ ┌──────────────────────┐ │
│ │ 心经        0 / 1 遍 │ │
│ │ 阿弥陀佛    36 /108  │ │
│ │ 大悲咒      未开始   │ │
│ └──────────────────────┘ │
│                          │
│ [开始今日功课]            │
│                          │
│ 最近阅读                 │
│ 金刚经 第十三品           │
│                          │
│ 今日一句                 │
│ 一切有为法，如梦幻泡影    │
└──────────────────────────┘
```

### 4.3 交互

- 点击“开始今日功课”：进入第一个未完成项目。
- 点击功课项目：进入对应阅读或计数页。
- 点击最近阅读：恢复阅读位置。
- 点击今日一句：进入原经文位置。

### 4.4 空状态

如果用户未设置功课：

- 显示“尚未设置今日功课”。
- 主按钮为“添加日课”。
- 推荐三个模板：入门、净土、简短早课。

```text
┌──────────────────────────┐

│ ┌──────────────────────┐ │
│ │ 搜索经名、译者、关键词 │ │
│ └──────────────────────┘ │
```

- 经名。
- 译者。
- 分类。
- 预计阅读时间。

- 是否有注释。

进入阅读前先展示：

- 经名。
- 简介。
- 译者。
- 来源。
- 版本说明。
- 章节目录。
- 阅读按钮。
- 加入日课按钮。

## 6. 阅读页

### 6.1 页面目标

阅读页是核心体验。正文必须是第一优先。

### 6.2 线框

```text
┌──────────────────────────┐
│ < 金刚经        目录  设置 │
│                          │
│ 如是我闻。一时佛在舍卫国， │
│ 祇树给孤独园，与大比丘众  │
│ 千二百五十人俱。          │
│                          │
│ 尔时，世尊食时，著衣持钵，│
│ 入舍卫大城乞食。          │
│                          │
│ ...                      │
│                          │
│ ┌──────────────────────┐ │
│ │ 00:42  ▶  第 12 / 86 段│ │
│ └──────────────────────┘ │
└──────────────────────────┘
```

### 6.3 底部工具栏

阅读状态：

- 目录。
- 播放。
- 书签。
- 显示设置。

播放状态：

- 暂停。
- 上一句。
- 下一句。
- 循环。
- 速度。

### 6.4 显示设置 Sheet

包含：

- 字号滑块。
- 简体/繁体分段控件。
- 拼音开关。
- 注释开关。
- 夜间模式开关。
- 自动滚动开关。

### 6.5 注释呈现

建议两种方式：

- 轻注释：段落下方小字号显示。
- 详细注释：点击术语后底部 Sheet 展开。

首版建议先做轻注释，避免交互复杂。

### 7.1 页面目标

提供念佛、诵咒、计数和音频入口。

### 7.2 线框

```text
┌──────────────────────────┐
│ 诵持                     │
│                          │
│ 今日诵持                 │
│ 阿弥陀佛  36 / 108        │
│ [继续]                   │
│                          │
│ 计数器                   │
│ ┌──────────────────────┐ │
│ │                      │ │
│ │         36           │ │
│ │                      │ │
│ │      [  记一声  ]     │ │
│ └──────────────────────┘ │
│                          │
│ 辅助                     │
│ 木鱼  引磬  静音          │
```

### 7.3 交互

- 自动计数模式需要明确开关，避免误计。

## 10. iOS 组件建议

### 10.1 SwiftUI 结构

```text
App
└─ AppShellView
   ├─ TodayStack
   ├─ LibraryStack
   ├─ ChantStack
   └─ ProfileStack
```

### 10.2 主要 View

- TodayView
- ScriptureLibraryView
- ScriptureDetailView
- ScriptureReaderView
- ReaderSettingsSheet
- ChantPracticeView
- CounterView
- DedicationView
- ProfileView

### 10.3 状态管理

- 当前 Tab：本地 State。
- 每个 Tab 的 NavigationPath：Tab 内持有。
- 阅读设置：共享设置服务。
- 经文数据：只读数据仓库。
- 用户记录：本地持久化服务。

### 10.4 Liquid Glass 使用建议

如果目标支持 iOS 26：

- 可用于底部迷你播放器。
- 可用于阅读设置 Sheet。
- 可用于今日页主按钮区域。

不要用于：

- 长正文背景。
- 经文段落卡片。
- 大面积阅读区域。

阅读可读性优先于视觉效果。

## 11. 首版设计验收标准

### 11.1 可读性

- 正文在 390pt 宽度下不拥挤。
- 默认字号适合中老年用户。
- 夜间模式不刺眼。
- 拼音和注释打开后不挤压正文。

### 11.2 操作效率

- 从首页开始今日功课不超过 2 次点击。
- 从经藏找到心经不超过 2 次点击。
- 阅读页打开设置不超过 1 次点击。
- 播放、暂停、书签随时可达。

### 11.3 克制感

- 无弹窗广告。
- 无排行榜。
- 无夸张动效。
- 无强迫连续打卡。
- 无商业化入口干扰诵读。
